为什么 t == nil, i == t, 而 i != nil???

综合了下众多回答，造成这个结果主要是：nil 接口与 nil 值差异造成的。

当执行 i == nil 的时候，由于 i 是接口类型，编译器会把没有类型的 nil 视为一个 interface{}类型的 nil 并与 i 进行对比。

在底层，interface作为两个成员实现：一个类型和一个值。该值被称为接口的动态值， 它是一个任意的具体值，而该接口的类型则为该值的类型。对于 int 值3， 一个接口值示意性地包含(int, 3)。

由于 i 初始化时等于 t，所以 接口 i 的类型为 T，而值为 nil，这与 nil 中 类型、值均为 nil 不符合，所以导致 i == nil 的结果为 false
